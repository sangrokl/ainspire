# 공모전 영상팀 — 워크플로 총괄 규칙

## 팀 구조

| 팀 | 에이전트 파일 | 역할 |
|----|--------------|------|
| 기획팀 | `.claude/agents/planning-team.md` | 요강 분석 → 기획안 |
| 생성팀 | `.claude/agents/production-team.md` | 스토리보드 → 이미지/HTML/영상 생성 |
| 검수팀 | `.claude/agents/review-team.md` | 결과물 점검 + 수정 우선순위 |

---

## 폴더 구조

```
공모전 영상팀/
├── CLAUDE.md
├── .claude/agents/
│   ├── planning-team.md
│   ├── production-team.md
│   └── review-team.md
├── 01_요강/
│   └── 요강분석.md           ← 기획팀 출력
├── 02_기획/
│   └── 기획안.md             ← 기획팀 출력
├── 03_스토리보드/
│   ├── 스토리보드.md         ← 생성팀 출력 (씬별 프롬프트 + 생성 로그)
│   └── 스토리보드.html       ← 생성팀 출력 (씬 이미지 갤러리)
├── 04_산출물/
│   ├── 이미지/               ← scene01.png, scene02.png …
│   └── 영상/                ← scene01.mp4, scene02.mp4 …
└── 05_검수/
    └── 검수보고서.md          ← 검수팀 출력
```

---

## 실행 순서

### STEP 1 — 기획팀

요강 URL을 전달해 기획팀 에이전트를 실행한다.

```
출력: 01_요강/요강분석.md, 02_기획/기획안.md
```

### STEP 2 — 생성팀

기획안을 전달해 생성팀 에이전트를 실행한다.
생성팀이 스토리보드 작성 후 씬 수·모델·예상 크레딧을 보고하면 승인 후 진행한다.
승인 시 "생성 진행 승인됨"을 프롬프트에 포함해 재호출하면 보고 없이 즉시 생성한다.

```
출력:
  03_스토리보드/스토리보드.md   (씬별 프롬프트 + 생성 로그)
  03_스토리보드/스토리보드.html (씬 이미지 갤러리)
  04_산출물/이미지/sceneNN.png
  04_산출물/영상/sceneNN.mp4
```

### STEP 3 — 검수팀

검수팀 에이전트를 실행한다. 입력 파일은 에이전트가 직접 읽는다.

```
출력: 05_검수/검수보고서.md
```

---

## 재실행 범위

| 상황 | 재실행 |
|------|--------|
| 기획 방향 수정 | STEP 1 → 3 |
| 씬 일부 재생성 | STEP 2 (해당 씬 지정) → STEP 3 |
| 검수 P1 수정 | STEP 2 (해당 씬 지정) → STEP 3 |
| 프롬프트만 조정 | STEP 2 → 3 |
